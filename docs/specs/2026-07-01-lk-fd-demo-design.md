# Личный кабинет FD — демо для портфолио. Дизайн-спека

**Дата:** 2026-07-01
**Статус:** утверждено к реализации
**Формат:** full-stack демо для портфолио

## 1. Контекст и цель

Демо-версия личного кабинета медиаселлера наружной рекламы (OOH) на основе
реального техзадания (`Личный кабинет FD Техзадание.xlsx`). Система агрегирует
рекламные конструкции разных Владельцев и даёт Клиенту-рекламодателю выбирать
рекламные поверхности: искать на карте/в списке, смотреть занятость по месяцам,
собирать рабочие списки, выгружать в Excel.

**Цель артефакта:** показать в портфолио **полный цикл full-stack разработки**
(модель данных, API, авторизация, интерактивный фронтенд, карта, экспорт,
админка, тесты, деплой). Демо должно работать «из коробки» по публичной ссылке
без ручной настройки ключей.

**Роли:**
- **Клиент** — работает в «Рабочем столе»: карта/список поверхностей, карточка
  стороны с занятостью по месяцам, рабочие списки, экспорт в Excel.
- **Админ** — управляет справочниками (владельцы, конструкции, поверхности,
  цены), импортирует фид, заводит клиентов.

Вход по логину/паролю. На странице входа — кнопки «Войти как Клиент / как Админ»
с предзаполненными демо-кредами (1 клик до обеих ролей).

## 2. Модель данных (Prisma / PostgreSQL)

- **Owner** (Владелец): `name, site, phone, email, contactPerson`.
- **Client** (Клиент): `name, site, phone, email, contactPerson`.
- **User**: `email, passwordHash, name, role (CLIENT | ADMIN), clientId?`.
- **Construction** (Конструкция): `ownerId, constructionNumber, ownerNumber,
  type, format, district, address, lat, lng, lighting, description, panoramaUrl`.
- **Surface** (Сторона): `constructionId, sideCode, direction, gid,
  surfaceNumber, photoUrl, mapPhotoUrl, grp, ots, esparId, oneShowSec,
  showsPerDay, material, printType, montage`.
- **Availability** (занятость по месяцам): `surfaceId, period (первое число
  месяца), status (FREE | SOLD | RESERVED_OTHER | NEEDS_CHECK), priceNet,
  priceGross`. Уникальность по `(surfaceId, period)`.
- **WorkingList** (Рабочий список): `clientId, name, createdAt`.
- **WorkingListItem**: `workingListId, surfaceId, addedAt`. Уникальность по
  `(workingListId, surfaceId)`.
- **FeedImport** (лог импорта): `fileName, importedAt, createdCount,
  updatedCount, ownerId?`.

Статусы занятости соответствуют легенде ТЗ: Свободно / Продано / Чужой резерв /
Необходимо уточнить наличие.

## 3. Архитектура и стек

- **Next.js 15** (App Router, TypeScript) — фронт и API в одном репозитории.
- **Tailwind CSS + shadcn/ui** — современная светлая SaaS-эстетика (типографика,
  мягкие тени, акцентный цвет, плотные но «дышащие» таблицы).
- **Prisma + PostgreSQL** — локально через docker-compose, деплой на Neon/Vercel
  Postgres. Переключение на SQLite — один шаг (документируется в README).
- **Auth.js (NextAuth v5)** — credentials-провайдер, роли, middleware защищает
  `/admin`.
- **API** — Next Route Handlers (`/api/...`), валидация входных данных **Zod**.
- **Таблицы** — TanStack Table (настройка видимости колонок, как в ТЗ).
- **Зоны рабочего стола** — `react-resizable-panels` (три независимые зоны,
  двигаются бегунками).
- **Состояние фильтров** — в URL (`searchParams`), чтобы выборка шарилась ссылкой.
- **Экспорт** — `exceljs`, генерация `.xlsx` на сервере.

## 4. Экраны

### Клиент — «Рабочий стол» (3 зоны, ресайз бегунками)

1. **Карточка стороны** — реквизиты поверхности (№ конструкции/владельца,
   владелец, тип, формат, координаты, свет, GRP/OTS, описание), фото и панорама,
   **календарь занятости по месяцам** (цвет = статус, цена в проданных/доступных),
   мини-форма «Ошибки, неточности» (сохраняется + тост).
2. **Карта / Список** (табы) + панель фильтров: Владелец, Район, Формат, Тип
   конструкции, Сторона, Период, Свободность.
   - **Карта** — react-leaflet, маркеры с кластеризацией, цвет по статусу, клик
     по маркеру открывает карточку стороны в зоне 1.
   - **Список** — таблица TanStack с настройкой видимости колонок; клик по строке
     открывает карточку.
3. **Рабочие списки** — вкладки списков: создать / переименовать / сохранить /
   удалить; добавление поверхностей по ID (вставка нескольких номеров); таблица
   позиций; «Загрузить на карту» (фильтрует карту по списку); «Выгрузить в Excel».

### Админ

- Справочник **владельцев** (CRUD).
- **Конструкции + стороны + цены** (CRUD; цены/статусы по месяцам).
- **Клиенты и пользователи** (CRUD).
- **Импорт фида** — загрузка CSV/XLSX нормализованного формата → создание/
  обновление поверхностей и занятости + запись в лог `FeedImport`.
- Мини-дашборд со статистикой (число конструкций, поверхностей, % занятости).

## 5. Слой карты (адаптер)

Интерфейс `MapProvider` с двумя реализациями:
- **LeafletProvider** (по умолчанию) — react-leaflet + OpenStreetMap +
  markercluster, **без API-ключа**. Демо всегда работает.
- **YandexProvider** — подгружает Yandex JS API 3.0, **если задан**
  `NEXT_PUBLIC_YANDEX_API_KEY`; добавляет панорамы.

Выбор провайдера — по наличию env-ключа. По умолчанию — OSM.

**Обоснование выбора карты:** 2ГИС отпадает (тайлы Map Tiles API платные,
демо-ключ живёт 1 месяц — не годится для постоянного портфолио). Яндекс JS API
3.0 реально бесплатен (25 000 запросов/день) но требует ключ, привязанный к
домену. OSM бесплатен и без ключа. Адаптер даёт лучшее из двух: работает у любого
зрителя на OSM, и опционально показывает «аутентичный» Яндекс с панорамами.

## 6. Демо-данные (seed)

Скрипт сидирования генерирует **~200 поверхностей по Санкт-Петербургу и области**
с реалистичными координатами, привязанные к владельцам из ТЗ (Билбордпост, Гриф,
РИМ, РУСС, ЭЛВИС, Реклама Центр, Перспектива, Леноблреклама, POSTEXX). Для каждой
поверхности — занятость и цены по месяцам 2026 с 4 статусами. Клиенты: ЛЭНЖИ, ДСК,
Ресторан Фокс, КАНТРИ ХАУС. Демо-пользователи: `client@demo` и `admin@demo`. Фото
и панорамы — плейсхолдеры.

## 7. Вне объёма (YAGNI для демо)

- Реальная отправка писем (форма ошибок только сохраняется + тост).
- Парсинг «экзотических» Excel каждого владельца (импорт — один нормализованный
  CSV/XLSX формат).
- Хостинг реальных фото/панорам.
- Оплата, договоры, бронирование с блокировкой.
- Точное повторение всех колонок каждого владельца.
- Мультиязычность (только русский).

## 8. Тестирование и деплой

- **Vitest** — юнит-тесты на бизнес-логику: расчёт/агрегация занятости, форматы
  цен, применение фильтров, парсинг вставки ID.
- **Playwright** — e2e smoke: вход → применение фильтра → добавление поверхности
  в рабочий список → экспорт в Excel.
- Деплой на **Vercel + Neon**. README с демо-кредами, инструкцией локального
  запуска (docker-compose Postgres, миграции, seed) и скриншотами.

## 9. Критерии готовности демо

- Публичная ссылка открывается, вход в 1 клик под обеими ролями.
- Карта показывает ~200 маркеров по СПб, фильтры меняют выборку карты и списка.
- Клик по маркеру/строке открывает карточку с календарём занятости.
- Можно создать рабочий список, добавить поверхности, выгрузить корректный .xlsx.
- Админка позволяет создать владельца/конструкцию/поверхность и импортировать фид.
- Проходят unit- и e2e-тесты; README воспроизводит запуск с нуля.
